Feedback Form

Technical Description

This project is a responsive and well-structured feedback form web page designed using HTML and CSS. It is intended to collect user feedback related to Sustainable Development Goal 15: Life on Land. The form is divided into two main sections: personal details and feedback details. In the personal section, users are asked to provide their full name, email address, telephone number, address, and date of birth. Input fields are validated using built-in HTML5 attributes such as required, type="email", and pattern for the phone number to ensure proper data entry. The feedback section includes various types of inputs such as radio buttons, checkboxes, and a star rating system. Users can share their familiarity with SDG 15, environmental concerns they care about, actions they’ve taken, and their opinion on who should lead in protecting life on land. A visual star-based rating allows users to indicate their satisfaction with the webpage, and a message box is provided for additional comments. The layout of the page is created using CSS Flexbox to position the left informational panel and the form container side by side. The form design features a green-themed color palette, rounded corners, and user-friendly spacing. Styling is applied to enhance the form's appearance, including custom placeholder colors, interactive hover effects, and visually distinct input sections. While the form provides an engaging and intuitive interface, it is front-end only and does not connect to a server or database for processing responses. Overall, the project demonstrates a clean, accessible, and environmentally themed user interface suitable for awareness and educational purposes.

Accessibility

This feedback form web page incorporates several accessibility features to ensure it is usable by a wide range of users, including those with disabilities. Semantic HTML elements such as form, fieldset, legend, label, and appropriate input types are used throughout the form, which enhances screen reader compatibility and provides better structure for assistive technologies. Team page

Technical Description

The Team Page is a responsive and visually engaging web page developed using HTML and CSS to introduce and highlight the individual contributions of each team member involved in a web development project. The layout consists of a central heading followed by four individual profile cards, each representing a student team member. Each profile card includes a professional avatar image, the member’s name, role, and a list of specific pages or sections they were responsible for implementing within the project. The names and roles are styled with custom colors and fonts to maintain consistency and readability. The design uses float and fixed-width boxes to align team member cards horizontally, with hover effects applied to reveal additional details within a sliding "under" section. This effect is created using CSS transitions and opacity changes, providing an interactive user experience. Each card is styled with borders, shadows, and rounded corners to enhance visual aesthetics. The images automatically scale to fit their containers, maintaining responsiveness across devices. The page also includes personalized color schemes to distinguish roles and contributions clearly. This page serves both as an acknowledgment of individual effort and as an organized way to display team structure, making it ideal for academic or collaborative project portfolios. Although the current implementation focuses solely on front-end design without dynamic functionality or scripting, it provides a strong visual representation and intuitive layout using only HTML and CSS.

Content Page

Technical Description

This webpage presents an informative overview of biodiversity through a structured and responsive design. It features a fixed sidebar navigation for easy access to key sections including About, Hotspots, Conservation, Threats, and Loss, enhancing user experience and site navigation. The content area uses semantic HTML elements such as headings, paragraphs, definition lists dl, dt, dd, and tables to organize information logically and improve readability. Images are incorporated with responsive styling to maintain visual balance across different screen sizes. CSS media queries enable a mobile-friendly layout by adjusting the sidebar to a top position on smaller screens. The page employs a clean and accessible color scheme with green accents to emphasize biodiversity topics, and consistent typography ensures clarity. Overall, the design prioritizes accessibility, responsiveness, and semantic structure, making complex ecological information easily digestible for a broad audience.

Accessibility

This webpage is designed with accessibility in mind to ensure that users of diverse abilities can access and understand the content effectively. Semantic HTML elements like headings h1, h2, h3, definition lists dl, dt, dd, and descriptive alt attributes on images provide meaningful structure and context for screen readers, improving navigation and comprehension for visually impaired users. The fixed sidebar uses clear and consistent link text for easy keyboard navigation, and the active and hover states have sufficient color contrast to aid users with visual impairments. The responsive design, implemented via CSS media queries, ensures the layout adapts smoothly across different devices and screen sizes, which benefits users who rely on screen magnifiers or use mobile devices. Text sizes are set with relative units, and color choices meet WCAG contrast guidelines to enhance readability. However, to further improve accessibility, ARIA landmarks (like nav or main), skip links, and keyboard focus styles could be added for enhanced usability and navigation.

Challenges and Lessons Learned

Creating the Team Page came with a few challenges that taught us important lessons. Aligning the profile boxes using CSS float required careful adjustments and helped us realize the benefits of modern layout techniques like Flexbox. Implementing the hover effect to reveal team details was visually appealing but also showed us the importance of making interactive elements accessible to all users, including those using keyboards or screen readers. Ensuring consistent styling across all cards improved our understanding of organized CSS and design consistency. Overall, this project improved our skills in HTML/CSS layout, styling, and accessibility awareness.

Compliance

The Team Page has been developed in compliance with standard web development practices, following the principles of semantic HTML and CSS styling. It adheres to basic accessibility guidelines by using proper HTML structure, descriptive image alt text, and readable color contrast. The layout and design elements are created using valid CSS without relying on JavaScript, ensuring compatibility across modern browsers. Although some improvements could enhance full WCAG compliance—such as better keyboard navigation and focus indicators—the page meets the general requirements for academic and portfolio-based web projects.
