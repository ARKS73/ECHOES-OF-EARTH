Profile Page

Technical Description

The profile page for Maya Fernando is a responsive, nature-inspired portfolio built with HTML5 and CSS, featuring a clean two-column grid layout (sidebar and main content) with flexbox for nested components. It includes interactive elements like hover effects, skill progress bars, and a color-coded availability calendar, all styled with an eco-friendly green palette. The design is fully responsive, with media queries adjusting the footer and layout for mobile, while maintaining accessibility and visual appeal through semantic structure and dynamic CSS features.

Accessibility

The profile page demonstrates strong accessibility with semantic HTML, keyboard navigation, sufficient color contrast, and labeled form inputs, achieving ~85% WCAG 2.1 AA compliance. Key strengths include proper heading hierarchy, focus states, and image `alt` attributes. Suggested improvements include adding ARIA landmarks, replacing placeholder text with explicit labels, implementing skip links, and enhancing calendar accessibility with `aria-labels` and non-color indicators. Focus states and screen-reader testing would further optimize the experience for all users.

Sitemap-Page

Technical Description

The sitemap is built using an interactive SVG diagram within an HTML5 structure, styled with CSS for a nature-inspired green theme. It features a responsive layout with dynamic SVG scaling, arrow markers for visual hierarchy, and organized node-link relationships. The design leverages semantic elements, inline styling for flexibility, and a clean, structured approach to site navigation mapping.

Accessibility

The sitemap demonstrates strong accessibility with semantic HTML, keyboard-navigable links (tabindex="0"), and ARIA labels for screen readers. It maintains sufficient color contrast and provides visual feedback via hover/focus states. However, adding skip links and explicit labels for form elements (if any) would further enhance accessibility.

Content Page

Technical Description

Built with a responsive CSS Flexbox layout (flex-direction: row-reverse), the page features a content-aside structure with eco-themed styling (#a5d6a7, #e8f5e9). It uses fixed positioning for the "Back to Top" button, semantic sectioning, and optimized images with relative sizing (width: 90%). The design prioritizes readability with clear typography and hierarchical headings, while interactive elements (links, hover states) maintain visual consistency with the green color palette.

Accessibility

The page demonstrates strong accessibility with proper heading hierarchy (h1-h3), descriptive alt text for images, and keyboard-navigable links. It maintains sufficient color contrast (#2e7d32/#f4f9f4) and includes a "Back to Top" button for easier navigation. While the current structure supports accessibility well, adding ARIA landmarks and skip links would further enhance compatibility with screen readers. The design effectively balances visual clarity with functional accessibility features. Challenges and Lessons Learned

This project taught us valuable lessons about merging accessibility with design. We initially struggled with coding structure and validation errors, which slowed progress until we implemented stricter code reviews. Choosing color schemes that met contrast requirements while preserving visual appeal required multiple adjustments. A key realization was that accessibility features like keyboard navigation need to be built into the initial design, not added later. Through trial and error, we learned that automated validation tools and early user testing prevent costly reworks. The experience highlighted how accessibility and aesthetics can work together through careful planning and attention to detail. Most importantly, we now understand that accessible design isn't restrictive – when done right, it enhances the experience for all users.

Compliance

This project deepened our understanding of web compliance standards, particularly WCAG 2.1 requirements. We initially underestimated how subtle design choices - like button contrast ratios and interactive element sizing - could impact compliance. Through rigorous testing, we identified and resolved multiple AA-level accessibility issues, including improving color contrast ratios from 3.5:1 to 4.5:1 for normal text. The process revealed that compliance isn't just about checking boxes; it requires thoughtful consideration of how different users interact with content. We implemented responsive design principles to ensure compatibility across devices while maintaining compliance. Documenting our decisions helped create a replicable framework for future projects. Most importantly, we learned that true compliance means building accessibility into every development phase rather than treating it as a final checklist.
